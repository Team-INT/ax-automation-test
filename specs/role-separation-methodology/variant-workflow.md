# Variant 워크플로우 가이드

> Variant 도구를 활용한 디자인 단계 (Step 2) 실행 가이드.
> `design-director` 에이전트가 이 가이드를 기반으로 사용자에게 작업 지시를 제공한다.

---

## 1. Variant란?

텍스트를 입력하면 UI 디자인을 자동으로 생성해주는 AI 디자인 도구. 디자인 탐색에 특화되어 있으며, 다양한 변형 디자인을 즉시 생성하고 코드로 내보낼 수 있다.

**핵심 기능:**
- 텍스트 → UI 디자인 자동 생성
- 스크롤하면 다양한 변형 (Variant) 제공
- 스타일 변경, 컬러 변경, 레이아웃 셔플
- 코드 내보내기 (Claude Code, Codex 등으로 직접 전달)
- "New Chat from Design" — 선택한 디자인의 톤을 이어받아 새 섹션 생성

---

## 2. 작업 순서

### 2.1 히어로 섹션 (톤 설정)

> 첫 섹션이 전체 사이트의 톤을 결정한다. 가장 신중하게 선택해야 한다.

**Variant 프롬프트 예시:**

```
다음 히어로 섹션을 강렬하고 임팩트 있는 톤으로 디자인해줘.

[copy.md의 Hero 섹션 전체 붙여넣기]

조건:
- 다크 배경 + 밝은 텍스트
- 헤드라인을 가장 크게
- CTA 버튼은 accent 컬러로 눈에 띄게
- 모던하고 프로페셔널한 느낌
```

**선택 기준:**
- 스크롤하며 여러 변형 확인
- Strong 값 조절로 강도 변경 가능
- 컬러, 레이아웃, 밀도가 "이 사이트의 톤"에 맞는지 판단
- 선택한 디자인이 다음 섹션의 **기준점**이 됨

### 2.2 후속 섹션 (체이닝)

> 히어로에서 설정한 톤을 이어받아 나머지 섹션을 만든다.

**프로세스:**

1. 히어로 디자인에서 **"New Chat from Design"** 클릭
2. 다음 섹션의 카피를 **반드시** 붙여넣기

**Variant 프롬프트 예시 (Problem 섹션):**

```
위 디자인과 같은 톤으로 다음 Problem 섹션을 디자인해줘.

[copy.md의 Problem 섹션 전체 붙여넣기]

조건:
- 위 히어로와 동일한 컬러/폰트/여백
- 3개 문제점을 카드 형태로
- 각 카드에 숫자를 accent 컬러로 강조
```

**Service → Testimonials → CTA 순서로 반복.**

### 2.3 주의사항

| 상황 | 올바른 방법 | 잘못된 방법 |
|------|-----------|-----------|
| 새 섹션 생성 | "New Chat from Design" 사용 | 처음부터 새로 생성 |
| 카피 전달 | copy.md 원문 **그대로** 붙여넣기 | "적당히 문구를 만들어줘" |
| 톤 유지 | 히어로 디자인 기반 체이닝 | 매 섹션 새로운 스타일 시도 |
| 스타일 조정 | Strong 값/컬러/레이아웃 셔플 | 프롬프트로 장황하게 설명 |

## 3. 코드 내보내기

### 3.1 Variant → Claude Code

1. 모든 섹션 디자인 완료 확인
2. 각 섹션에서 **"Open in"** 버튼 클릭
3. **"Claude Code"** 선택
4. 생성된 코드 복사
5. Claude Code 데스크탑 앱에 붙여넣기

### 3.2 내보내기 프롬프트 (Claude Code에 전달)

```
아래 5개 섹션의 디자인 코드를 기반으로 웹사이트를 구현해줘.

[섹션 1~5 Variant 코드 붙여넣기]

조건:
1. 위 코드에서 디자인 시스템(컬러, 폰트, 간격)을 추출하여 Tailwind @theme 변수로 구축
2. 텍스트는 copy.md 파일의 내용으로 교체
3. 이미지는 브랜드 에셋으로 교체 (경로: public/images/)
4. 브랜드 폰트 적용 (next/font)
5. Next.js + Tailwind CSS로 반응형 구현
6. 모바일 우선 (Mobile-first)
```

## 4. Figma 대안 워크플로우

Variant 대신 Figma를 사용하는 경우:

### 4.1 Figma MCP 연동

`design-director` 에이전트는 Figma MCP 도구를 통해 디자인 컨텍스트를 직접 가져올 수 있다:

```
get_design_context  — 디자인 노드의 코드 + 스크린샷 + 컨텍스트 힌트
get_screenshot      — 디자인 스크린샷
get_metadata        — 파일/노드 메타데이터
```

### 4.2 Figma → design-spec.md

1. Figma에서 디자인 작업 (copy.md 기반)
2. Figma URL을 `/enf:design-guide`에 전달
3. 에이전트가 Figma MCP로 디자인 정보 추출
4. `design-spec.md` 자동 생성

### 4.3 Figma 프롬프트 예시

```
다음 Figma 디자인을 분석하여 design-spec.md를 생성해줘.

Figma URL: https://figma.com/design/{fileKey}/{fileName}?node-id={nodeId}
기반 카피: copy.md

위 디자인에서:
1. 컬러 팔레트 추출 → CSS 변수로
2. 타이포그래피 추출 → CSS 변수로
3. 여백/간격 추출 → @theme 토큰으로
4. 각 섹션의 레이아웃, 인터랙션 명세
```

## 5. 디자인 검수 체크리스트

Variant/Figma 작업 완료 후 다음을 확인:

- [ ] **톤 일관성**: 히어로와 CTA를 나란히 놓았을 때 "같은 사이트"인가?
- [ ] **카피 무결성**: copy.md 원문이 그대로 사용되었는가? (AI가 변경하지 않았는가?)
- [ ] **섹션 완전성**: 5개 섹션 모두 디자인되었는가?
- [ ] **컬러 통일**: 전 섹션에서 동일한 컬러 팔레트를 사용하는가?
- [ ] **폰트 통일**: 전 섹션에서 동일한 폰트 패밀리를 사용하는가?
- [ ] **CTA 가시성**: CTA 버튼이 각 섹션에서 눈에 띄는가?
- [ ] **이미지 영역**: 이미지 교체가 필요한 영역이 명확하게 식별되는가?

통과 시 → `design-spec.md` 작성 후 `/enf:design-implement`로 진행.
