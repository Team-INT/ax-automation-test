# copy.md 템플릿 (Quick Reference)

> 상세 버전: `specs/role-separation-methodology/copy-template.md`

## 구조

```markdown
# Copy: {프로젝트명}

> 생성일: {날짜}
> 타겟: {타겟 고객 요약}
> 서비스: {서비스명}

## 고객 리서치

### 타겟 페르소나
{JSON: name, age, role, company_size, goals, pain_points, search_keywords, desired_outcome}

### 경쟁 분석 요약
{테이블: 경쟁사, 핵심 메시지, 약점}

## 섹션 1: Hero
- **공감 질문**: 타겟의 고충을 질문 형태로
- **헤드라인**: 핵심 가치 제안 + 구체적 숫자
- **서브카피**: 1-2문장 보충
- **CTA**: 구체적 행동 + 허들 낮추기

## 섹션 2: Problem
- **헤드라인**: 문제의 심각성 + 숫자
- **문제점 목록**: 3개 (각각 상황 + 수치)
- **전환 문구**: 문제 → 솔루션 연결

## 섹션 3: Service
- **헤드라인**: 솔루션 한 줄 요약
- **특징 목록**: 3개 (특징명 + 설명 + 효과 수치)
- **보조 CTA**: 중간 전환점

## 섹션 4: Testimonials
- **후기 2-3개**: 이름/직함 + 구체적 경험 인용문 + 전후 수치

## 섹션 5: CTA
- **헤드라인**: 마지막 설득
- **CTA 텍스트**: 허들 낮추기
- **보조 텍스트**: 불안 해소
- **신뢰 뱃지**: 신뢰 요소 나열
```

## 핵심 규칙

| DO | DON'T |
|----|-------|
| 구체적 숫자 (73%, 47건) | "혁신적인", "다양한" |
| 타겟 실제 상황 | 범용적 메시지 |
| CTA 허들 낮추기 | "시작하세요" |
| 후기 구체적 경험 | 모호한 칭찬 |
| 전후 수치 비교 | 주관적 표현 |
