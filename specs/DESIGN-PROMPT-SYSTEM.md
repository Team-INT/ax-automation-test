## 1. 시스템 개요

### 1.1 이 문서의 사용법

이 시스템은 **3단계 레이어**로 구성됩니다:

```
Layer 1: 컨텍스트 주입    → Claude에게 프로젝트 배경을 이해시킴
Layer 2: 디자인 지시      → 구체적인 디자인 요구사항 전달
Layer 3: 품질 검증        → 생성된 결과물을 벤치마크 기준으로 검증
```

### 1.2 프롬프트 구조 원칙

모든 디자인 프롬프트는 다음 5가지 요소를 포함해야 합니다:

| 요소 | 설명 | 예시 |
|------|------|------|
| **WHO** | 타겟 사용자 | "기업 CSR/ESG 담당자 (35세, 대기업 과장)" |
| **WHAT** | 만들어야 할 것 | "기업 파트너십 랜딩 페이지" |
| **WHY** | 비즈니스 목적 | "파트너십 문의 전환율 향상" |
| **HOW** | 디자인 스펙 | "컨셉 A 스타일, Deep Forest + Gold 컬러" |
| **REFERENCE** | 벤치마크 기준 | "Acumen의 3분화 CTA 패턴 참조" |

### 1.3 금지 사항

```
[X] "예쁘게 만들어줘" — 주관적, 비구체적
[X] "모던하게" — 해석이 다양함
[X] "적당히 배치" — 기준 없음
[X] 레퍼런스 없이 요청 — 디자이너 재량에 의존

[O] "Kiva 히어로 섹션처럼 4대 임팩트 수치를 카운터 애니메이션으로"
[O] "Acumen의 Support/Partner/Grow 3분화 CTA 구조로"
[O] "charity: water 스타일의 스크롤 드리븐 데이터 시각화"
```

## 2. 프로젝트 컨텍스트 주입

### 2.1 기본 컨텍스트 (모든 프롬프트 앞에 붙임)

아래 블록을 Claude 대화 시작 시 **시스템 프롬프트 또는 첫 메시지**로 전달합니다:

```markdown
## 프로젝트 컨텍스트

### 기관 정보
- 기관명: 재단법인 굿네이버스 글로벌 임팩트 (Good Neighbors Global Impact Foundation)
- 슬로건: "Empowering People, Transforming Community"
- 사업 규모: 26개국 / 621개 협동조합 / 13개 사회적기업 / 5개 소액금융기관(MFI)
- 4대 사업: 협동조합 육성, 소액금융, 사회적기업 설립 지원, 소셜비즈니스
- 자체 브랜드: 더네이버스커피 (르완다·과테말라 직거래 커피), 메리쿱 (감성 제품)

### 타겟 오디언스 (우선순위)
1. **1순위**: 글로벌 파트너 기관 + 기업 후원사 (CSR/ESG 담당자)
   - 원하는 것: 임팩트 데이터, 파트너십 사례, 사업소개서 PDF
   - 사이트 역할: 파트너십 확대, 기업후원 유치
2. **2순위**: 개인 후원자 (일반 기부자)
   - 원하는 것: 후원 방법, 임팩트 시각화, 간편 결제
   - 사이트 역할: 개인후원 전환 동선 확보
3. **3순위**: 내부 직원 + 채용 대상자
   - 원하는 것: JD, 조직 문화, 지원 프로세스

### 사이트 톤
- 기관/기업 대상의 **신뢰감 있는 전문적 톤** 중심
- 데이터와 근거 기반의 임팩트 증명
- 개인후원 동선도 확보 (감성적 연결 포함)

### 다국어
- 한국어 + 영문 (핵심 4개 섹션만: About, Programs, Impact, Partnership)
- URL: /ko/, /en/ prefix 구조

### 핵심 문제 (현재 사이트)
1. 전환 동선 부재 — 기업후원 단일 페이지만 존재
2. 임팩트 비가시 — 메인에 성과 수치 없음
3. 개인후원 불가 — 개인 기부 경로 전무
```

## 8. 프롬프트 조합 전략

### 8.1 단계별 진행 순서

디자인 프롬프트는 다음 순서로 진행합니다:

```
Phase 1: 공통 컴포넌트
├── GNB 네비게이션
├── 푸터
├── 수혜자 스토리 카드
├── 임팩트 카운터
└── CTA 버튼 시스템

Phase 2: 메인 페이지
├── 히어로 섹션
├── 4대 사업 카드
├── 스토리 캐러셀
├── 3분화 CTA
├── 파트너 로고 캐러셀
└── 뉴스레터 CTA

Phase 3: 핵심 서브 페이지
├── 임팩트 대시보드 (/impact)
├── 기업 파트너십 (/support/corporate)
├── 개인 후원 (/support/donate)
└── 사업 개요 + 글로벌 맵 (/programs)

Phase 4: 보조 서브 페이지
├── 재단소개 (/about)
├── 개별 사업 페이지 (/programs/cooperative 등)
├── 소식/뉴스룸 (/news)
├── 채용 (/careers)
└── 기타 (이용약관, 개인정보)

Phase 5: 인터랙션 & 정제
├── 스크롤 애니메이션 통합
├── 반응형 테스트/수정
├── 접근성 검증
└── 성능 최적화
```

"수혜자 인용문을 영문으로도 표시해줘. 한국어 아래에 영문 번역을 13px italic으로."

"카운터 애니메이션 시간을 2초에서 3초로 늘리고,
easeOutExpo 대신 easeInOutCubic 커브로 변경해줘."

### 10.3 자주 하는 실수 & 해결

| 실수 | 문제 | 해결 |
|------|------|------|
| 컨텍스트 누락 | Claude가 일반적인 디자인 생성 | 항상 섹션 2.1 컨텍스트 먼저 주입 |
| "예쁘게" 요청 | 주관적 해석, 브랜드 불일치 | 구체적 스펙 (컬러 코드, px 값, 레퍼런스) 명시 |
| 한 번에 전체 페이지 | 품질 저하, 세부사항 누락 | 섹션 단위로 분리 요청 |
| 레퍼런스 미지정 | 디자인 방향 불명확 | 벤치마크 사이트 + 구체적 패턴 명시 |
| 모바일 미고려 | 데스크톱만 생성 | 반응형 브레이크포인트 명시 포함 |
| 접근성 미고려 | WCAG 미준수 | 섹션 7.2 체크리스트 포함 |
| 피드백 두루뭉술 | 수정 불명확 | "~를 ~로 변경" 구체적 지시 |

### 10.4 프롬프트 난이도별 추천

| 난이도 | 대상 | 추천 방식 |
|--------|------|----------|
| 초급 | 디자인 경험 거의 없음 | 섹션 4~5의 완성된 프롬프트 복사 사용 |
