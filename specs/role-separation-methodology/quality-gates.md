# 품질 게이트 (Quality Gates)

> 역할 분리 방법론의 각 단계 전환 시 적용되는 품질 검증 기준.
> Gate를 통과해야 다음 단계로 진행할 수 있다.

---

## Gate 1: 카피 검증 (Copy → Design)

`/enf:copy-review` 커맨드로 자동 실행.

### 필수 통과 항목 (5개 모두 PASS 필요)

#### 1.1 구체성 (Specificity)

| 기준 | PASS | FAIL |
|------|------|------|
| Hero 헤드라인에 구체적 숫자 포함 | "73% 절감" | "획기적으로 절감" |
| Problem 섹션에 수치 기반 문제 제시 | "매주 8시간 소요" | "많은 시간 소요" |
| Service 특징에 효과 수치 포함 | "87% 시간 절감" | "빠르게 처리" |
| Testimonials에 전후 비교 수치 | "8시간 → 30분" | "많이 줄었습니다" |
| CTA에 구체적 시간/조건 명시 | "30분이면 알 수 있습니다" | "시작하세요" |

**검사 방법**: 각 섹션에서 숫자(`\d+`)가 최소 1개 이상 존재하는지 확인

#### 1.2 완전성 (Completeness)

| 기준 | PASS | FAIL |
|------|------|------|
| 5개 섹션 모두 존재 | Hero + Problem + Service + Testimonials + CTA | 섹션 누락 |
| 고객 리서치 섹션 존재 | 페르소나 JSON + 경쟁 분석 | 리서치 없이 카피만 |
| Hero에 공감 질문 + 헤드라인 + 서브카피 + CTA | 4개 항목 모두 | 항목 누락 |
| Testimonials에 최소 2개 후기 | 2개 이상 | 1개 이하 |
| CTA에 신뢰 뱃지 포함 | 있음 | 없음 |

#### 1.3 페르소나 정합성 (Persona Alignment)

| 기준 | PASS | FAIL |
|------|------|------|
| 페르소나의 pain_point가 Problem 섹션에 반영 | 1:1 매핑 확인 | 리서치와 카피 불일치 |
| 페르소나의 role/context가 Hero에 반영 | 타겟 언어 사용 | 범용적 메시지 |
| 검색 키워드가 카피에 자연스럽게 포함 | 키워드 존재 | 키워드 부재 |

#### 1.4 CTA 명확성 (CTA Clarity)

| 기준 | PASS | FAIL |
|------|------|------|
| 구체적 행동 동사 사용 | "데모 보기", "체험 시작" | "시작하세요", "알아보기" |
| 허들 낮추기 정보 포함 | "카드 등록 없이", "5분" | 조건 없음 |
| 중간 CTA (Service 섹션) 존재 | 보조 CTA 있음 | 최종 CTA만 있음 |

#### 1.5 톤 일관성 (Tone Consistency)

| 기준 | PASS | FAIL |
|------|------|------|
| 문체 통일 | 전체 해요체 또는 전체 합니다체 | 섹션별 문체 변화 |
| 감정 온도 일관 | 전체 동일 톤 | Hero는 강렬, CTA는 차분 등 |
| 브랜드 보이스 반영 | 일관된 어조 | 섹션마다 다른 성격 |

### 판정

- **PASS**: 5개 항목 모두 통과 → Step 2 진행
- **PARTIAL**: 1-2개 항목 미흡 → 해당 항목 수정 후 재검증
- **FAIL**: 3개 이상 미흡 → Step 1 재실행 권장

## Gate 2: 디자인 검증 (Design → Implementation)

사용자가 직접 검수. `/enf:design-guide` 실행 후 체크리스트로 제공.

### 필수 통과 항목 (4개 모두 PASS 필요)

#### 2.1 톤 잠금 (Tone Lock)

| 기준 | PASS | FAIL |
|------|------|------|
| 전체 섹션 컬러 팔레트 통일 | 히어로 팔레트와 동일 | 섹션마다 다른 컬러 |
| 폰트 패밀리 통일 | 히어로와 동일 heading/body | 섹션마다 다른 폰트 |
| 여백/밀도 일관 | 섹션 간격, 카드 패딩 통일 | 들쭉날쭉한 간격 |
| 레이아웃 톤 일관 | 동일한 그리드 시스템 | 섹션마다 다른 구조 철학 |

**체크 방법**: 히어로와 마지막 섹션(CTA)을 나란히 놓고 "같은 사이트인가?" 확인

#### 2.2 카피 무결성 (Copy Integrity)

| 기준 | PASS | FAIL |
|------|------|------|
| 히어로 헤드라인이 copy.md 원문과 동일 | 글자 하나 차이 없음 | AI가 임의 변경 |
| 모든 섹션의 카피가 copy.md 기반 | 원문 유지 | 새로운 카피 생성됨 |
| CTA 텍스트가 copy.md 원문과 동일 | 글자 하나 차이 없음 | "시작하기"로 변경됨 |

**체크 방법**: `copy.md`와 Variant 출력의 텍스트를 1:1 대조

#### 2.3 반응형 계획 (Responsive Plan)

| 기준 | PASS | FAIL |
|------|------|------|
| Mobile 레이아웃 명시 | 1컬럼, 터치 최적화 | Desktop 전용 |
| Tablet 레이아웃 명시 | 2컬럼 | 미정의 |
| 브레이크포인트 표 존재 | 4단계 정의 | 미정의 |
| 타이포 스케일 (Desktop/Mobile) 명시 | 크기 차이 정의 | 단일 크기만 |

#### 2.4 토큰 추출 (Token Extraction)

| 기준 | PASS | FAIL |
|------|------|------|
| 컬러가 CSS 변수로 정의 | `--color-primary: #xxx` | `#hex`만 흩어져 있음 |
| 폰트가 CSS 변수로 정의 | `--font-heading: "..."` | 폰트명만 텍스트로 |
| @theme 블록 존재 | Tailwind 적용 가능 | CSS 변수 없음 |
| 인터랙션 기본값 정의 | duration, easing 명시 | "부드럽게" 같은 표현 |

### 판정

- **PASS**: 4개 항목 모두 통과 → Step 3 진행
- **PARTIAL**: 1개 항목 미흡 → 해당 항목 보완 후 진행
- **FAIL**: 2개 이상 미흡 → Step 2 재실행 (Variant 재작업)

## Gate 3: 구현 검증 (Implementation Complete)

기존 enf 커맨드 조합으로 실행.

### 검증 명령어

```bash
/enf:code-review          # 코드 품질 검사
/enf:type-check           # TypeScript 타입 검증
/enf:perf-audit           # 성능 분석
```

### 필수 통과 항목

| 기준 | 검증 방법 | 통과 기준 |
|------|----------|----------|
| 카피 반영 | copy.md와 렌더링 텍스트 비교 | 100% 일치 |
| 디자인 토큰 적용 | globals.css의 @theme 확인 | design-spec.md 토큰 모두 적용 |
| 반응형 | 브라우저 기기 전환 테스트 | 4단계 브레이크포인트 정상 |
| 성능 | Lighthouse | Performance 90+ |
| 접근성 | Lighthouse + 키보드 테스트 | Accessibility 90+ |
| 타입 안전성 | /enf:type-check | 에러 0건 |

## 피드백 루프

### 실패 시 돌아갈 단계

| 실패 지점 | 원인 | 돌아갈 단계 |
|----------|------|------------|
| Gate 1 구체성 FAIL | 카피가 추상적 | Step 1 (카피 재작성) |
| Gate 1 페르소나 FAIL | 리서치 부족 | Step 1 (리서치부터 재실행) |
| Gate 2 카피 무결성 FAIL | Variant가 카피 변경 | Step 2 (해당 섹션만 재작업) |
| Gate 2 톤 잠금 FAIL | 체이닝 실패 | Step 2 (히어로부터 재체이닝) |
| Gate 3 카피 반영 FAIL | 구현 시 카피 누락 | Step 3 (카피 교체만 재실행) |
| Gate 3 성능 FAIL | 이미지/번들 과대 | Step 3 (최적화) |
