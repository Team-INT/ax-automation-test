# copy.md 템플릿

> `/enf:copy-write` 커맨드의 출력 형식.
> `copy-strategist` 에이전트가 이 형식에 맞춰 카피를 생성한다.

---

## 사용법

1. `/enf:copy-write "서비스 설명 + 타겟 고객"` 실행
2. 에이전트가 아래 형식의 `copy.md` 파일을 생성
3. 사용자가 검수 후 `/enf:copy-review`로 품질 검증
4. 통과 시 `/enf:design-guide`의 입력으로 사용

---

## 템플릿

```markdown
# Copy: {프로젝트/페이지명}

> 생성일: {날짜}
> 타겟: {타겟 고객 요약}
> 서비스: {서비스/제품명}

## 섹션 1: Hero

- **공감 질문**: {타겟의 고충을 질문 형태로}
  - 예: "광고 세팅에만 하루 8시간, 정작 전략은 언제 짜시나요?"
- **헤드라인**: {핵심 가치 제안, 구체적 숫자 포함}
  - 예: "광고 운영 시간 73% 절감 — AI가 세팅부터 리포트까지"
- **서브카피**: {1-2문장, 헤드라인 보충}
  - 예: "세팅, 최적화, 리포트를 자동화해서 당신은 전략에만 집중하세요."
- **CTA**: {구체적 행동 + 허들 낮추기}
  - 예: "5분 데모 보기 — 카드 등록 없이 바로 시작"

## 섹션 2: Problem (문제 제기)

- **헤드라인**: {문제의 심각성을 숫자로 보여주기}
  - 예: "마케터의 업무 시간 중 62%가 반복 작업에 소비됩니다"
- **문제점 목록**:
  1. {문제 1 + 구체적 상황/수치}
     - 예: "광고 세팅: 채널별 반복 세팅에 매주 8시간"
  2. {문제 2 + 구체적 상황/수치}
     - 예: "리포트 작성: 엑셀 수작업으로 월 3일 소비"
  3. {문제 3 + 구체적 상황/수치}
     - 예: "A/B 테스트: 복잡한 세팅 때문에 아예 안 하게 됨"
- **전환 문구**: {문제에서 솔루션으로 연결}
  - 예: "이 시간을 전략과 크리에이티브에 쓸 수 있다면?"

## 작성 원칙

### DO (이렇게 쓴다)

- 구체적인 **숫자**를 넣는다 (73%, 47건, 30분)
- 타겟의 **실제 상황**을 반영한다 (매주 월요일 오전, 엑셀 수작업)
- CTA에 **허들을 낮추는** 정보를 포함한다 (카드 등록 없이, 5분 데모)
- 후기에 **구체적 경험**을 담는다 (커피 한 잔 마시는 동안)
- 전후 비교를 **수치로** 보여준다 (8시간 → 30분)

### DON'T (이렇게 쓰지 않는다)

- "혁신적인", "최고의", "다양한" 같은 **추상적 수식어** 사용 금지
- "시작하세요", "지금 바로" 같은 **뻔한 CTA** 금지
- 경쟁사를 **직접 언급**하거나 비방 금지
- 증명할 수 없는 **과장된 수치** 사용 금지
- 모든 섹션에 **같은 톤** 유지 (갑자기 격식체 ↔ 반말 전환 금지)
